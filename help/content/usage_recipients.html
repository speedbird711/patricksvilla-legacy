SimpleSecure Usage: Recipient Records

Typical form processors allow you to specify the email recipient as a hidden variable in the form, or else you must configure recipient in the script itself.

SimpleSecure uses a slightly different approach that allows for both flexibility and security.  When you create a new HTML form, you also create a corresponding "Recipient" Record for that form.  The Recipient Record allows you to specify to whom the message will be sent, encryption settings, etc.

On your HTML form, you include a hidden variable with name="cfgId".  The value refers to the Recipient Record Id.  When SimpleSecure processes the form, it will use the settings specified for the provided Recipient Record.

One Recipient Record can support an unlimited number of HTML forms, so long as the intended receiver of the formmail message is the same.  You need an additional Recipient Record for each actual person/email to which a form will be sent.

For Single-Site installation, this means that you will probably only need to configure one (1) recipient record.  If you are hosting multiple sites, or providing secure form processing for various people, you simply create a Recipient Record for each client.  You provide that client with thier Recipient Record ID.  Then, they may create as many forms as they like, so long as they use the correct cfgId on their form.

Recipient Record & Form Settings:

The following fields are available for each Recipient Record & Form.  Those setting that have an HTML Fieldname can be specified on the HTML form itself.  The default setting for the Recipient Record will be used unless an override setting is specified on the form.

Any HTML form field that begins with the letters 'cfg' will be considered a configuration setting and will not be included in the email message.


Setting HTML FieldName Description
Id cfgId This value is incremented automatically.  Include a hidden variable 'cfgId' on your form with this Id to specify the RecipientRecord.
FormName   Enter a friendly name to identify this form.
EncryptMethod   Specifies if this form data should be encrypted. 
EncryptKey   If you specified an encryption type that uses public key cryptography, this is the public key to use when encrypting.
Recipients   Addresses to which the formmail will be sent.  (One email address per line)
AttNameLookup   If you specified an encrytion type that is sent as an attachment, a hidden or visible form field can be used to control the filename of the attachment. If none specifed, the file will be given a semi-random numeric name. Suggestion: cfgFrom
RequireSSL   If Yes is selected, SimpleSecure will not process the form unless it is submitted though an SSL connection. Recommended to select Yes as a precaution if this is a secure form with encryption.
DefaultFrom cfgFrom Email address from which the form submission will be emailed.
DefaultSubject cfgSubject Subject of the form submission email message. Can be overridden with the hidden form variable 'cfgSubject'
DefaultRedirectUrl cfgRedirectUrl Upon successful submission, the visitor will be redirected to this page. If not specified, a generic thank-you message will be displayed.
DefaultEmailFormat cfgEmailFormat If the message is encrypted, it is recommended to use TEXT format.
HtmlTableStyle   If email is HTML format, Enter CSS code here to control the style of the table. (Use TABLE, TH and TD tags)
Auto-Reply   Specify if an auto-reply is sent. (configured in the Auto-Reply section)
The fields below are available only on the HTML form:
  cfgRequired comma-separated list of required form fields. (ex. "Name,Age,Phone")
  cfgSupress any non-empty value will tell SimpleSecure to ignore blank fields (fields that the user did not fill out will not be included in the email message).
